# REFERIX PRODUCT GUIDELINES

**Status:** Final
**Date:** 2024-07-09
**Stage:** XII — Product Excellence

---

## 1. Filosofía

Referix no es un sistema administrativo. Es una herramienta de trabajo diario. Cada pantalla debe sentirse profesional, rápida y natural, como Stripe, Linear o Notion.

## 2. Principios

| Principio | Significado |
|---|---|
| **Simplicidad** | Menos es más. Cada elemento debe justificarse. |
| **Claridad** | El usuario entiende en <5s qué hacer. |
| **Velocidad** | Cada acción debe sentirse instantánea. |
| **Elegancia** | Espacio, tipografía, animaciones sutiles. |
| **Consistencia** | Misma experiencia en las 8 aplicaciones. |
| **Confianza** | Nada parpadea, nada se mueve, nada sorprende. |

## 3. Acción Principal

Cada pantalla tiene UNA acción principal. Nunca dos.

| App | Acción Principal |
|---|---|
| Sales | "Nueva venta" |
| Console | "Gestionar operación" |
| Partner | "Solicitar pago" |
| Customer 360 | "Buscar cliente" |
| Executive | "Ver resumen" |

## 4. La Prueba de los 5 Segundos

Toda pantalla debe responder en <5s:

1. ¿Qué estoy viendo?
2. ¿Qué puedo hacer aquí?
3. ¿Cuál es la info más importante?
4. ¿Qué acción espera el sistema?
5. ¿Qué pasa si no hago nada?

## 5. Colores

| Color | Significado |
|---|---|
| 🔵 Azul | Acción, información |
| 🟢 Verde | Correcto, éxito, comisión |
| 🟡 Amarillo | Atención, pendiente |
| 🔴 Rojo | Error, requiere acción urgente |

## 6. Información por Prioridad

No mostrar todo primero. Mostrar lo importante:

1. Contexto (¿para quién es?)
2. Beneficio (¿qué resuelve?)
3. Dato técnico (velocidad, precio)
4. Detalle (documentos, plazos)

## 7. Microinteracciones

Toda acción debe tener feedback inmediato:
- Click → respuesta visual en <100ms
- Acción completada → animación sutil
- Error → mensaje claro + acción sugerida

## 8. Cero Fricción

- Reducir clics a la mitad
- Completar automáticamente campos predecibles
- No preguntar lo que el sistema ya sabe
- Una acción = un click, no un formulario
